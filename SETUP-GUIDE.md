# 🚀 دليل الإعداد الكامل - خطوة بخطوة

## ❌ المشكلة: الخطوات فاضية؟

**السبب:** لم يتم تنفيذ السكريبت في قاعدة بيانات Supabase بعد!

---

## ✅ الحل - اتبع هذه الخطوات بالترتيب:

### الخطوة 1️⃣: إنشاء مشروع Supabase (إذا لم يكن موجوداً)

1. اذهب إلى: https://supabase.com
2. سجّل دخول أو أنشئ حساب جديد
3. اضغط **New Project**
4. املأ البيانات:
   - **Name:** marai-alshimal
   - **Database Password:** اختر كلمة سر قوية (احفظها!)
   - **Region:** اختر أقرب منطقة (مثل: EU West)
5. اضغط **Create new project**
6. ⏳ انتظر 1-2 دقيقة حتى يجهز المشروع

---

### الخطوة 2️⃣: تنفيذ السكريبت في قاعدة البيانات

1. **في Supabase Dashboard:**
   - من القائمة الجانبية، اختر **SQL Editor**

2. **افتح التبويب الجديد:**
   - اضغط **New query**

3. **انسخ محتوى الملف:**
   - افتح ملف `supabase/schema.sql` من المشروع
   - انسخ **كل المحتوى** (Ctrl+A ثم Ctrl+C)

4. **الصق في SQL Editor:**
   - الصق المحتوى في نافذة SQL Editor

5. **نفّذ السكريبت:**
   - اضغط زر **Run** (أو F5)
   - ⏳ انتظر بضع ثوانٍ

6. **تحقق من النجاح:**
   - يجب أن ترى رسالة **"Success. No rows returned"**
   - ✅ هذا يعني تم إنشاء الجداول بنجاح!

---

### الخطوة 3️⃣: نسخ معلومات الاتصال

1. **في Supabase Dashboard:**
   - اذهب إلى: **Settings** → **API**

2. **انسخ هذه القيم:**

### الخطوة 4️⃣: إنشاء ملف .env.local

1. **في مجلد المشروع `marai-alshimal`:**
   - انسخ ملف `.env.local.example`
   - غيّر اسمه إلى `.env.local`

2. **افتح `.env.local` وعدّل القيم:**

```env
NEXT_PUBLIC_SUPABASE_URL=https://xxxxxxxxxxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

3. **احفظ الملف** (Ctrl+S)

---

### الخطوة 5️⃣: إعادة تشغيل الخادم

1. **أوقف الخادم الحالي:**
   - في Terminal، اضغط `Ctrl+C`

2. **شغّل الخادم من جديد:**
   ```bash
   npm run dev
   ```

3. **انتظر حتى يبدأ:**

   ```
   ✓ Ready in 2.5s
   ```

## 🔍 التحقق من البيانات

افتح في المتصفح:
```
http://localhost:3000/api/admin/step-fields?step=1
```

**النتيجة المتوقعة:**
```json
{
  "data": [
    {
      "id": "...",
      "step": 1,
      "field_name": "hashi_incoming_count",
      "field_label": "🐪 حاشي - الكمية",
      "field_type": "number",
      "is_required": true,
      ...
    },
```

**إذا رأيت `"data": []` (فارغ):**
❌ السكريبت لم يُنفّذ في Supabase - ارجع للخطوة 2

## 🆘 حل المشاكل

### المشكلة 1: "لا توجد حقول محددة لهذه الخطوة"

**السبب:** السكريبت لم يُنفّذ في Supabase

**الحل:**
1. ارجع للخطوة 2
2. تأكد من تنفيذ **كل** محتوى `schema.sql`
3. تحقق من ظهور رسالة "Success"

---

### المشكلة 2: "Supabase connection error"

**السبب:** معلومات الاتصال في `.env.local` خاطئة

**الحل:**
1. راجع الخطوة 3 و 4
2. تأكد من نسخ القيم **بالكامل** (بدون مسافات زائدة)
3. أعد تشغيل الخادم (Ctrl+C ثم `npm run dev`)

---

### المشكلة 3: الخطوات تظهر لكن فاضية

**السبب:** جدول `step_fields` موجود لكن بدون بيانات

**الحل:**
1. في Supabase → **Table Editor**
2. افتح جدول `step_fields`
3. إذا كان فارغاً:
   - احذف الجدول
   - أعد تنفيذ السكريبت من الخطوة 2

---

### المشكلة 4: "relation 'step_fields' does not exist"

**السبب:** الجدول غير موجود في قاعدة البيانات

**الحل:**
1. نفّذ السكريبت كاملاً من الخطوة 2
2. **لا تنفّذ أجزاء منه** - نفّذه كاملاً دفعة واحدة
