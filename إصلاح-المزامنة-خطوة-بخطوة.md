# 🔧 إصلاح مشكلة المزامنة — خطوة بخطوة

## 🔴 المشكلة
عند الضغط على "مزامنة الآن" أو "سحب تاريخ مخصص" يظل التايمر يعمل بدون ما يسحب أي بيانات.

## 🟡 السبب
البرنامج الصغير المسؤول عن سحب البيانات من جهاز Aronium (**السكريبت**) غير مثبت أو متوقف على **كمبيوتر الكاشير**.

---

## ✅ الخطوات — اتبعها بالترتيب

---

### 📌 الخطوة 1 — روح على كمبيوتر الكاشير (الجهاز اللي فيه برنامج Aronium)

> ⚠️ **مهم**: الخطوات التالية تتم على كمبيوتر الكاشير، **مش** على جهازك أنت

---

### 📌 الخطوة 2 — افحص إذا السكريبت مثبت أصلاً

افتح مستكشف الملفات (Windows Explorer) وروح على المسار:
```
C:\AroniumSync
```

**إذا المجلد موجود** ← روح للخطوة 4

**إذا المجلد مو موجود** ← لازم تثبت السكريبت من الصفر، روح للخطوة 3

---

### 📌 الخطوة 3 — تثبيت السكريبت من الصفر (إذا ما كان مثبتاً)

**أ)** انسخ مجلد `aronium-sync` بالكامل من هذا المشروع إلى كمبيوتر الكاشير
(المجلد موجود في: `c:\Users\Mohammed\OneDrive\Desktop\Alshamal\marai-alshimal\aronium-sync`)

**ب)** على كمبيوتر الكاشير، افتح الملف الذي نسخته:
```
aronium-sync\config.json
```

**ج)** عدّل الملف وضع البيانات الصحيحة:
```json
{
  "branch_id":        "UUID الفرع من Supabase",
  "supabase_url":     "https://XXXX.supabase.co",
  "supabase_key":     "eyJhbG...",
  "aronium_db_path":  "C:\\Users\\اسم-المستخدم\\AppData\\Local\\Aronium\\Data\\pos.db",
  "branch_name":      "مراعي الشمال 3"
}
```

> 💡 **كيف تعرف branch_id؟** افتح لوحة الإدارة → الفروع → الفرع اللي تريده ← انسخ الـ ID من الرابط

**د)** انقر بالزر الأيمن على `install.ps1` ← اختر **"Run with PowerShell"**
أو افتح PowerShell كـ Administrator واكتب:
```powershell
powershell -ExecutionPolicy Bypass -File "C:\AroniumSync\install.ps1"
```

**هـ)** اتبع التعليمات واملأ البيانات المطلوبة

---

### 📌 الخطوة 4 — إذا المجلد موجود لكن المزامنة ما تشتغل

#### 4أ) شغّل ملف التشخيص

في مجلد `C:\AroniumSync` ← ابحث عن ملف اسمه:
```
تشخيص-المزامنة.bat
```

إذا ما وجدته، انسخه من هنا:
```
c:\Users\Mohammed\OneDrive\Desktop\Alshamal\marai-alshimal\aronium-sync\تشخيص-المزامنة.bat
```

**شغّل الملف بـ double-click** — سيفتح نافذة سوداء وينفذ هذه الخطوات تلقائياً:
1. ✅ يتحقق من Python
2. ✅ يتحقق من sync.py وconfig.json
3. ✅ يتحقق من مهمة الجدولة (Task Scheduler)
4. 🔄 يشغّل مزامنة فورية الآن
5. 📄 يعرض آخر سطور من السجل

**اقرأ ما يظهر في النافذة:**

---

#### 4ب) إذا ظهر خطأ "Python غير مثبت"

ثبّت Python مجاناً من الرابط:
```
https://www.python.org/downloads/
```
> ⚠️ عند التثبيت ضع علامة صح على "Add Python to PATH"

بعد التثبيت شغّل ملف التشخيص مرة ثانية

---

#### 4ج) إذا ظهر خطأ "مهمة الجدولة غير مثبتة"

هذا يعني السكريبت لا يعمل تلقائياً. لإعادة تثبيت مهمة الجدولة:

1. افتح **PowerShell** كـ Administrator (انقر يمين على Start → Windows PowerShell (Admin))
2. اكتب:
```powershell
powershell -ExecutionPolicy Bypass -File "C:\AroniumSync\install.ps1"
```
3. إذا سألك عن بيانات الفرع، أدخلها مرة أخرى

---

#### 4د) تحقق يدوي إذا السكريبت شغّال

افتح **Command Prompt** (ابحث عن `cmd` في Start) واكتب:

```cmd
cd C:\AroniumSync
python sync.py --trigger
```

إذا اشتغل بنجاح ستشاهد رسائل مثل:
```
[INFO] فحص طلبات المزامنة...
[INFO] وجدت 1 طلب مزامنة!
[INFO] اكتملت المزامنة: 45 فاتورة
```

بعدها ارجع للداشبورد، المزامنة يجب أن تنجح ✅

---

### 📌 الخطوة 5 — تأكيد النجاح

بعد تنفيذ الخطوات السابقة:

1. ارجع للداشبورد على المتصفح
2. روح على **مبيعات POS** ← اختر الفرع
3. سيظهر شريط **أخضر** مكتوب فيه "السكريبت يعمل"
4. اضغط "مزامنة الآن" — يجب أن تكتمل خلال دقيقة أو أقل

---

## 🆘 إذا ما حل شيء

أرسل لي محتوى ملف السجل:
```
C:\AroniumSync\sync.log
```

افتحه بـ Notepad وأرسل لي آخر 20 سطر فيه، وسأحدد المشكلة بدقة.

---

## 📋 ملخص سريع

| الحالة | الحل |
|--------|------|
| مجلد `C:\AroniumSync` غير موجود | ثبّت السكريبت: شغّل `install.ps1` |
| Python غير مثبت | حمّل Python من python.org |
| مهمة الجدولة غير موجودة | أعد تشغيل `install.ps1` |
| كل شيء موجود لكن ما يشتغل | شغّل `تشخيص-المزامنة.bat` وأرسل النتيجة |
