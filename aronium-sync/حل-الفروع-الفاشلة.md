## 🔴 الأسباب الشائعة للفشل

### السبب الأول (الأكثر شيوعاً): مسار pos.db خاطئ
عند التثبيت، المسار الافتراضي هو:
```
C:\Users\[اسم-المستخدم]\AppData\Local\Aronium\Data\pos.db
```
إذا تم التثبيت بحساب مستخدم مختلف عن حساب تشغيل Aronium، المسار سيكون خاطئاً.

**مثال:**
- تم التثبيت على حساب `Admin` → المسار = `C:\Users\Admin\AppData\Local\...`
- لكن Aronium مثبت على حساب `Cashier` → الملف فعلياً في `C:\Users\Cashier\AppData\Local\...`

---

## ✅ خطوات الإصلاح (على كل جهاز فاشل)

### الخطوة 1: شغّل أداة التشخيص
```
انقر مزدوج على: تشخيص-وإصلاح-شامل.bat
```
ستخبرك بالمشكلة تحديداً.

---

### الخطوة 2: إذا كانت المشكلة "مسار DB خاطئ"
```
انقر مزدوج على: إصلاح-مسار-DB.bat
```
ستبحث تلقائياً عن pos.db في كل مكان وتُصلح المسار.

---

### الخطوة 3: إذا كانت المشكلة "Python غير مثبت"
1. حمّل Python من: https://python.org/downloads
2. **مهم جداً**: ضع علامة صح على **"Add Python to PATH"**
3. ثم شغّل `تثبيت-كمسؤول.bat` من جديد

---

### الخطوة 4: إذا كانت المشكلة "المهمة غير مسجلة"
```
انقر يمين على: تثبيت-كمسؤول.bat → تشغيل كمسؤول
```

---

## 🔍 كيف تعرف أن الإصلاح نجح؟

افتح `C:\AroniumSync\sync.log` وابحث عن:
```
✓ اكتملت المزامنة: X فاتورة
```

أو راقب من لوحة الإدارة → **مزامنة POS** وتحقق أن آخر مزامنة تحدّثت.

---

## 📋 ملخص الأسباب المحتملة

| السبب | الحل |
|-------|------|
| مسار pos.db خاطئ | شغّل `إصلاح-مسار-DB.bat` |
| Python غير مثبت | ثبّت Python مع PATH ثم أعد التثبيت |
| المهمة غير مسجلة | شغّل `تثبيت-كمسؤول.bat` |
| branch_id خاطئ | تحقق من UUID الفرع في لوحة الإدارة → الفروع |
| لا يوجد إنترنت | تحقق من الاتصال |

---

## 🆘 إذا لم تنجح الحلول

شارك محتوى `C:\AroniumSync\sync.log` كاملاً لتحديد المشكلة بدقة.
