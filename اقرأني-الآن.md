# 🚨 حل مشكلة التقارير - خطوات واضحة ومباشرة

## المشكلة
- التقارير **موجودة** في قاعدة البيانات (8 تقارير)
- لكن الداش بورد يعرض **3 تقارير فقط**
- السبب: **Row Level Security (RLS)** في Supabase

---

## ✅ الحل (خطوة بخطوة)

### الخطوة 1: افتح Supabase Dashboard
1. اذهب إلى: https://supabase.com/dashboard
2. اختر مشروعك
3. من القائمة الجانبية، اختر **SQL Editor**

### الخطوة 2: نفذ السكربت
1. افتح الملف: **`FINAL-FIX.sql`**
2. انسخ **كل** المحتوى
3. الصقه في SQL Editor في Supabase
4. اضغط **Run** أو **Execute**

### الخطوة 3: انتظر النتائج
يجب أن ترى:
```
✅ ALTER TABLE
✅ ALTER TABLE  
✅ ALTER TABLE
✅ ALTER TABLE
✅ DO
✅ GRANT ALL...
✅ NOTIFY
```

وفي النهاية جدول يعرض:
```
report_date    | count
---------------+-------
2026-04-16    | 4
2026-04-15    | 4
```

### الخطوة 4: حدّث الداش بورد
1. ارجع لصفحة الداش بورد: http://localhost:3000/dashboard
2. اضغط **Ctrl + Shift + R** (أو Cmd + Shift + R في Mac) لإعادة تحميل كامل
3. **يفترض الآن تظهر كل التقارير!**

## 💡 إذا لم يحل المشكلة

إذا نفذت SQL ولا تزال المشكلة موجودة:

### 1. تحقق من تنفيذ SQL
ارجع لـ Supabase SQL Editor ونفذ:
```sql
SELECT tablename, rowsecurity FROM pg_tables 
WHERE schemaname = 'public' AND tablename = 'daily_reports';
```

النتيجة المتوقعة:
```
tablename      | rowsecurity
---------------+-------------
daily_reports | false        ← يجب أن تكون false
```

### 2. تحقق من عدد التقارير
نفذ هذا في SQL Editor:
```sql
SELECT COUNT(*), report_date 
FROM daily_reports 
GROUP BY report_date 
ORDER BY report_date DESC;
```

النتيجة المتوقعة:
```
count | report_date
------+-------------
  4   | 2026-04-16
  4   | 2026-04-15
```

### 3. أعد تشغيل السيرفر
في Terminal:
1. اضغط `Ctrl + C` لإيقاف السيرفر
2. شغّل مرة أخرى: `npm run dev`
3. افتح الداش بورد في نافذة خاصة (Incognito/Private)
